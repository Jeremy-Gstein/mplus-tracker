MPTracker API

REST API for tracking Mythic+ keystones run by guild members. Queries Raider.IO for character and guild data, stores runs in SQLite, and exposes counts and history for external consumers (Discord bots, dashboards, WeakAuras).

Base URL: http://localhost:8080
Format: application/json
Auth: ⚠ not set
Storage: SQLite (WAL)
Authentication
Bearer token required. Every request (except GET /health) must include an Authorization header. Requests without a valid token receive 401 Unauthorized.
Header format
Authorization: Bearer <your API_TOKEN>

Generate a token on the server with:
openssl rand -hex 32

Set it as API_TOKEN in your .env file next to docker-compose.yml. Use the sidebar or the chip above to configure it here for live Try It requests.
Example curl with auth
curl -H 'Authorization: Bearer <token>' 'http://localhost:8080/debug/stats'
Errors & Common Concepts

All error responses use the same envelope. The scope parameter controls the time window for all query endpoints and is region-aware for weekly resets.

Error envelope — every non-2xx response returns:
{ "error": "Missing `realm` parameter", "code": 400 }
Scope values
ValueWindowNotes
todayUTC midnight → nowCalendar day in UTC
weekLast weekly reset → nowNA: Tue 09:00 America/New_York (DST-aware) · EU: Wed 07:00 UTC. Pass region=eu for EU reset.
alltimeEpoch → nowDefault when scope is omitted
customfrom → toRequires both from and to as RFC3339 strings
Deduplication

Every run is assigned a deterministic hash derived from region, realm, character name, dungeon, level, timestamp, and source run ID. INSERT … ON CONFLICT (hash) DO NOTHING means re-crawling a character never creates duplicate rows.

Stale-crawl skip

When updating a guild, members whose Raider.IO last_crawled_at is within the last 3 months are skipped for individual key updates (still upserted for roster tracking). This dramatically reduces rate-limit pressure on large guilds.

Depleted run filtering

Runs with num_keystone_upgrades = 0 (depleted / over-time) are rejected at ingest and never written to the database. All counts reflect timed completions only.

Updates

Update endpoints pull fresh data from Raider.IO and persist it to the local database. All updates are idempotent — re-running them only inserts genuinely new runs.

POST /update/guild Sync guild roster from Raider.IO ▶

Fetches the guild profile from Raider.IO, upserts all members into the characters table. Members crawled recently by Raider.IO are upserted but their individual key fetch is skipped to conserve rate-limit budget.

Query parameters
ParamTypeRequiredDescription
regionstringrequiredWoW region — us eu kr tw
realmstringrequiredRealm name, e.g. Stormrage
namestringrequiredGuild name, e.g. Seems Good
200 OK400 Bad Request500 Server Error
Response schema
request_idstring (uuid)Unique ID for this request
guild.region/realm/namestringEchoed from input
members_addedintegerNew characters inserted this run
members_updatedintegerExisting characters refreshed
members_skipped_freshintegerMembers skipped — crawled by RIO recently
Example request
curl -X POST -H 'Authorization: Bearer <token>' \ 'http://localhost:8080/update/guild?region=us&realm=Stormrage&name=Seems%20Good'
POST /update/character Sync one character's recent runs from Raider.IO ▶

Fetches mythic_plus_recent_runs for the given character from Raider.IO and inserts any new timed runs into the database. Depleted runs (num_keystone_upgrades = 0) and duplicates (by hash) are silently skipped.

Query parameters
ParamTypeRequiredDescription
regionstringrequiredWoW region
realmstringrequiredCharacter realm
namestringrequiredCharacter name
200 OK400 Bad Request500 Server Error
Response schema
request_idstring (uuid)
character.region/realm/namestring
runs_insertedintegerNew timed runs written to DB
runs_ignoredintegerDepleted or duplicate runs skipped
rate_limitedbooleanAlways false — retry handled internally
Example request
curl -X POST -H 'Authorization: Bearer <token>' \ 'http://localhost:8080/update/character?region=us&realm=Stormrage&name=Duhva'
POST /update/all Sync all known characters concurrently ▶
⚠ High load: Triggers a Raider.IO fetch for every character in the database, bounded by the semaphore concurrency limit in config.toml. On large guilds this takes several minutes. Characters returning 400 "not found" are automatically pruned from the database.
200 OK500 Server Error
Response schema
request_idstring (uuid)
total_charactersintegerTotal characters attempted
updated_okintegerSuccessfully fetched + synced
failedintegerCharacters that errored (non-400)
prunedintegerCharacters auto-deleted (RIO 400 not found)
errorsstring[]Error messages for non-pruned failures
Example request
curl -X POST -H 'Authorization: Bearer <token>' http://localhost:8080/update/all

GET /guild/:region/:realm/:name/roster All characters linked to a guild in the local DB ▶

Returns every character in the local database whose guild_name and guild_realm match the requested guild. Run POST /update/guild to sync it from Raider.IO first.

200 OK

Management
DEL /character/:region/:realm/:name Remove a character and all their runs from the DB ▶

Permanently deletes a character row and all associated runs via cascade. Returns 404 if the character isn't in the database. Characters that return 400 from Raider.IO during POST /update/all are pruned automatically — this endpoint is for manual removal.

200 OK404 Not Found
Response schema
deletedbooleanAlways true on 200
region / realm / namestringEchoed from path
Example request
curl -X DELETE -H 'Authorization: Bearer <token>' \ 'http://localhost:8080/character/us/Stormrage/Showdo'
